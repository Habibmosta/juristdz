───────────────────────────────────────────────────────────────
⚠️ POURQUOI C'EST UN PROBLÈME?
───────────────────────────────────────────────────────────────

Vite.js (votre outil de build) fonctionne ainsi:

1. Variables AVEC "VITE_" → Exposées au frontend ✅
   Exemple: VITE_SUPABASE_URL → Accessible dans le code

2. Variables SANS "VITE_" → Ignorées/Cachées ❌
   Exemple: SUPABASE_URL → Non accessible dans le code

Avoir les deux peut causer des conflits et des erreurs!

───────────────────────────────────────────────────────────────
🔧 SOLUTION IMMÉDIATE
───────────────────────────────────────────────────────────────

ÉTAPE 1: Aller sur Vercel
→ vercel.com
→ Votre projet "juristdz"
→ Settings
→ Environment Variables

ÉTAPE 2: Supprimer 2 Variables

Variable 1 à supprimer:
  Nom: SUPABASE_URL
  Action: Cliquez ••• → Delete → Confirmer

Variable 2 à supprimer:
  Nom: SUPABASE_ANON_KEY
  Action: Cliquez ••• → Delete → Confirmer

ÉTAPE 3: Vérifier
Vous devriez avoir UNIQUEMENT des variables VITE_*

ÉTAPE 4: Redéployer (déjà fait automatiquement)
Le push que nous venons de faire va redéployer l'app.
